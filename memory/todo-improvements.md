## 详细说明

### P0 - 急需（影响核心体验）

#### 1. 通知系统
**场景：**
- 新作业发布时通知学生
- 作业被批改时通知学生
- 日报被评论时通知学生
- 新文档上传时通知相关用户

**实现要点：**
- 数据库添加 notifications 表
- 后端 API：获取通知列表、标记已读、发送通知
- 前端：通知铃铛组件、通知列表页面
- 可选：集成邮件发送服务

#### 2. 日报评论/反馈功能
**场景：** 学生提交日报后，老师可以逐条点评

**实现要点：**
- 数据库：添加 daily_report_comments 表
- 后端 API：添加评论、获取评论列表
- 前端：日报详情页显示评论区域
- 显示"已阅"状态和老师的简短评语

#### 3. 搜索与筛选功能
**场景：** 数据多了以后难以查找

**实现要点：**
- 作业列表：按标题、状态（已提交/未提交）、截止时间筛选
- 文档列表：按分类、标题搜索
- 日报列表：按日期范围、内容关键词搜索
- 用户列表：按用户名、角色筛选

---

### P1 - 重要（提升使用体验）

#### 4. 富文本编辑器
**场景：** 日报和作业描述目前都是纯文本，不支持图片内嵌、代码块等

**实现要点：**
- 推荐：WangEditor 或 Toast UI Editor
- 支持图片内嵌、代码块、公式等
- 特别对于技术类作业，代码高亮是刚需

#### 5. 草稿与自动保存
**场景：** 填写日报或作业时，误刷新页面会导致内容丢失

**实现要点：**
- 本地存储草稿（localStorage）
- 定期自动保存（每30秒）
- 提交前确认提示
- 适用于：日报、作业提交、作业创建

#### 6. 文件在线预览
**场景：** 目前文件只能下载后查看，体验差

**实现要点：**
- 图片：直接预览
- PDF：PDF.js 预览
- Office文档：使用微软或金山预览API
- 视频：HTML5 播放器

---

### P2 - 优化（锦上添花）

#### 7. 学习数据可视化
**场景：** 目前数据都是列表展示，缺少趋势分析

**实现要点：**
- 日报提交热力图（类似 GitHub contributions）
- 作业成绩趋势图
- 学习时长统计
- 使用 ECharts 或 Chart.js

#### 8. 移动端适配
**场景：** 当前页面在移动端体验可能不佳

**实现要点：**
- 优化表格在移动端的展示（卡片式布局）
- 文件上传支持手机拍照上传
- 优化触摸交互

#### 9. 批量操作功能
**场景：** 老师管理大量学生作业时效率低

**实现要点：**
- 批量下载作业（打包成 ZIP）
- 批量导出成绩
- 批量设置作业状态
- 全选/反选功能

#### 10. 数据备份机制
**场景：** 目前缺少自动备份，误删数据无法恢复

**实现要点：**
- 定期自动备份数据库 + 文件存储
- 关键操作（删除、批量修改）添加二次确认
- 操作日志记录

#### 11. 作业截止提醒
**场景：** 学生容易错过作业截止时间

**实现要点：**
- 截止前 24 小时、1 小时提醒
- 逾期自动标记但仍允许补交（标记为 late）
- 仪表盘显示即将截止的作业

#### 12. 日报模板功能
**场景：** 学生每天写日报不知道格式，质量参差不齐

**实现要点：**
- 支持配置日报模板
- 默认模板：今日学习内容、遇到的问题、明日计划
- 常用语快捷插入
- 模板管理（管理员）
