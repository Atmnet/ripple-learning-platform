## 1. 产品概述

### 1.1 产品定位

Ripple Learning Platform 是一款面向教育机构的**学员日常管理平台**，旨在帮助教师（管理员）和学生高效完成日常学习任务的管理、提交和批改工作。平台以「涟漪」为设计理念，寓意每位学员的学习进步都能如涟漪般扩散、影响周边。

### 1.2 目标用户

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| **管理员 (Admin)** | 机构教师或教务人员 | 管理系统所有功能，包括用户管理、作业发布、成绩管理、文档管理等 |
| **学生 (Student)** | 平台注册学员 | 提交日报、提交作业、查看成绩、下载文档 |

### 1.3 核心价值

- **简化流程**：一站式完成日报提交、作业布置与批改、文档分享
- **角色分明**：管理员与学生权限严格分离，保障数据安全
- **文件管理**：支持多文件上传，本地/云端存储灵活切换
- **进度追踪**：实时查看提交状态，掌握整体学习进度

## 2. 功能模块

### 2.1 用户认证模块

#### 2.1.1 功能列表

| 功能 | 描述 | 访问权限 |
|------|------|----------|
| 用户登录 | 输入用户名和密码，验证身份后返回 JWT Token | 公开 |
| 用户注册 | 新用户注册账号（默认角色为 student） | 公开 |
| 当前用户信息 | 获取登录用户的基本信息和角色 | 需认证 |

#### 2.1.2 技术实现

- **认证方式**：JWT (JSON Web Token)
- **密码存储**：bcrypt 加密存储
- **Token 有效期**：默认 7 天（可配置）
- **默认管理员账号**：`admin` / `password`

#### 2.1.3 业务流程

```
用户 → 登录页 → 输入账号密码 → 后端验证 → 生成 JWT → 前端存储 Token → 跳转首页
```

### 2.3 作业管理模块

#### 2.3.1 功能列表

| 功能 | 描述 | 访问权限 |
|------|------|----------|
| 查看作业列表 | 浏览所有作业（支持分页，按截止日期排序） | 公开 |
| 查看作业详情 | 查看单条作业的完整信息 | 公开 |
| 提交作业 | 上传作业文件并提交 | 需认证 |
| 创建作业 | 管理员创建新作业 | 管理员 |
| 修改作业 | 管理员修改作业信息 | 管理员 |
| 删除作业 | 管理员删除作业 | 管理员 |
| 批改作业 | 管理员对学生提交进行评分和反馈 | 管理员 |
| 查看提交列表 | 查看某个作业的所有提交 | 需认证 |
| 查看全部提交状态 | 管理员查看所有学生的提交/未提交状态 | 管理员 |
| 我的提交 | 查看自己的所有作业提交记录 | 需认证 |

#### 2.3.2 数据模型

```typescript
// 作业表 (assignments)
{
  id: number;           // 主键
  title: string;        // 作业标题
  description: string;  // 作业描述
  deadline: Date;       // 截止日期
  file_url: string;     // 作业附件（JSON 数组格式）
  created_by: number;   // 创建者 ID
  created_at: Date;
  updated_at: Date;
}

// 作业提交表 (assignment_submissions)
{
  id: number;           // 主键
  assignment_id: number; // 关联作业 ID
  user_id: number;      // 提交用户 ID
  file_url: string;     // 提交文件（JSON 数组格式）
  comment: string;      // 提交备注
  grade: number;        // 分数
  feedback: string;     // 教师反馈
  graded_by: number;    // 批改人 ID
  submitted_at: Date;   // 提交时间
  graded_at: Date;      // 批改时间
}
```

#### 2.3.3 业务流程

```
【管理员】
创建作业 → 设置标题、描述、截止时间、上传附件 → 发布

【学生】
浏览作业 → 查看详情 → 截止日前提交 → 上传文件/填写备注 → 提交成功

【批改流程】
管理员查看提交 → 下载学生作业文件 → 评分 + 反馈 → 提交批改结果
```

### 4.8 响应格式

- **成功响应**：
  - 列表：`{ data: [...], total: number }`
  - 单条：直接返回实体对象
- **错误响应**：`{ error: "错误信息" }`
- **HTTP 状态码**：
  - 200：成功
  - 201：创建成功
  - 400：请求错误
  - 401：未认证
  - 403：无权限
  - 404：资源不存在
  - 500：服务器错误

## 5. 数据库设计

| 表名 | 描述 | 主键 | 外键 |
|------|------|------|------|

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 3 秒 |
| API 响应时间 | < 1 秒（正常负载） |
| 并发用户数 | 支持 100+ 用户同时在线 |
| 文件上传大小 | 最大 50MB |

### 6.2 安全要求

| 项目 | 要求 |
|------|------|
| 密码加密 | 使用 bcrypt 加密，salt rounds ≥ 10 |
| JWT Secret | 必须使用强密钥，禁止硬编码 |
| API 鉴权 | 所有敏感接口需携带有效 JWT Token |
| 角色权限 | 管理员接口需验证 role = 'admin' |
| 输入验证 | 前后端均需进行输入校验，防止注入攻击 |
| 文件安全 | 限制上传文件类型，防止恶意文件 |

### 6.3 兼容性要求

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| 屏幕适配 | 支持桌面端（1920×1080）及平板端 |
| 移动端 | 暂不支持，未来可考虑响应式布局 |

### 6.4 可用性要求

| 项目 | 要求 |
|------|------|
| 系统运行时间 | 7×24 小时运行 |
| 数据备份 | 每日自动备份（生产环境） |
| 错误处理 | 友好的错误提示，不暴露内部信息 |

### 6.5 技术栈要求

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3.5+, TypeScript 5.9+, Vite 7.3+, Element Plus 2.3+, Pinia 2.1+ |
| 后端 | Node.js 18+, Express 4.18+, TypeScript 5.3+ |
| 数据库 | MySQL 8.0 |
| 文件存储 | 百度云 BOS（支持本地回退） |
| 部署 | Docker / PM2 |

## 7. 已知问题与改进建议

基于代码审查，发现以下问题需在后续迭代中改进：

### 7.1 安全问题

| 问题 | 严重程度 | 建议 |
|------|----------|------|
| BOS 凭证硬编码 | 高 | 迁移至环境变量或密钥管理服务 |
| 默认 JWT Secret | 高 | 使用强随机密钥，禁止默认值 |
| 密码重置逻辑 | 中 | 实现邮件验证码而非明文密码 |

### 7.2 代码质量问题

| 问题 | 严重程度 | 建议 |
|------|----------|------|
| 过多 `as any` 类型断言 | 中 | 完善类型定义，减少类型断言 |
| console.log 残留 | 低 | 替换为结构化日志框架 |
| 缺少单元测试 | 中 | 引入 Jest/Vitest 编写测试 |

### 7.3 前端问题

| 问题 | 严重程度 | 建议 |
|------|----------|------|
| Vue ref 模板使用错误 | 高 | 检查所有模板中 ref 的使用 |
| 缺少加载状态 | 中 | 添加 skeleton 或 loading 状态 |

### 7.4 架构改进

| 改进项 | 描述 |
|--------|------|
| 日志框架 | 引入 Winston 或 Pino 替代 console.log |
| API 文档 | 使用 Swagger/OpenAPI 生成文档 |
| 单元测试 | 覆盖率目标 60%+ |
| 错误监控 | 接入 Sentry 等监控服务 |
